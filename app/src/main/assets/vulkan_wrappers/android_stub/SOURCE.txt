Bionic Vulkan Wrapper runtime libraries

Wrapper source:
https://github.com/leegao/bionic-vulkan-wrapper

Wrapper package/runtime reference:
https://github.com/leegao/vulkan_wrapper_termux-packages/tree/dev/wrapper

Bundled runtime libraries:
- libadrenotools.so from leegao/vulkan_wrapper_termux-packages wrapper/usr/lib/libadrenotools.so
- libc++_shared.so from Termux package libc++_29_aarch64.deb

libcutils.so is a local compatibility shim for legacy property_get and atrace
symbols used by the wrapper. property_get forwards to Android bionic
__system_property_get; atrace entry points are no-op stubs.

libhardware.so is a local compatibility shim for hw_get_module. The wrapper
declares this legacy dependency, but the Android WSI path does not need HAL
module loading.

libnativewindow.so is a local compatibility shim for AHardwareBuffer symbols.
It forwards calls to the platform libandroid.so at runtime.

The Android WSI wrapper build does not use the original Termux X11/DRM runtime
stack. Vulkan WSI is forwarded directly to the Android Vulkan driver.

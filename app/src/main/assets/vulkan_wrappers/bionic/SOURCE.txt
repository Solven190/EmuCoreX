The upstream ETC2-Milestone-2 wrapper advertises VK_KHR_android_surface but does
not implement the Android WSI entrypoints. This bundled build forwards Android
surface and swapchain calls to the underlying Vulkan driver while keeping the
wrapper path separate from the normal Vulkan renderer and the custom Adreno
driver path. It also forwards requested instance extensions to the underlying
driver before creating the wrapped instance, so Android WSI entrypoints can be
resolved after vkCreateInstance. Surface destruction is treated as externally
owned on the wrapper path to avoid Android loader surface double-free during
failed initialization cleanup. The Android build also forwards VK_KHR_swapchain
to the underlying driver, because the wrapper's Android WSI path delegates
swapchain creation and presentation to the platform Vulkan implementation. It
also tolerates foreign images in CmdPipelineBarrier and forwards patched image
barriers to avoid null wrapper-image lookups during emulator texture layout
transitions. Surface destruction is forwarded during normal lifecycle cleanup;
the app side avoids the failed-initialization double-free path separately.
